Phase A: CVLPSsが少ない → CVLP（CV系LP）への流入を改善
A
まずCVLPに十分なセッションを集めることが最優先
CVRを議論する前に、母数となるCVLP（TOP/カテゴリ/一覧ページ）へのSsが不足している状態。SEO施策でオーガニック流入を拡大する。
▼ 確認ポイントと施策
1
カテゴリ / 一覧ページのKW獲得状況
各CVLPがターゲットKWで順位を取れているか確認する
確認: ターゲットKWの順位 GSCで主要カテゴリKWの平均掲載順位を確認。20位圏外が多い場合は下記施策
確認: KWカバー率 想定されるカテゴリKW数に対して、実際に順位を獲得しているKWの割合

施策

  • SEOカテゴリページのtitle / H1にターゲットKWを最適配置
  • SEOカテゴリ説明文の追加・充実（200文字以上のユニークテキスト）
  • SEO下層カテゴリ・関連カテゴリへの内部リンク設計
  • SEOパンくずリスト / BreadcrumbList構造化データ整備
▼
2
TOPページからCVLPへの導線設計
TOPページに流入はあるが、CVLPへ遷移していないケース
確認: TOP→カテゴリ遷移率 GA4でTOPページからカテゴリ一覧への遷移率を確認
確認: グローバルナビの利用率 カテゴリナビのクリック率が低い場合はナビ設計の問題

施策

  • SEO+CVRファーストビューに主要カテゴリへのビジュアルリンク配置
  • SEO+CVRメガメニュー / カテゴリナビの改善
  • SEOTOPページのtitle / metaにブランド+主要カテゴリKW
CVLPSsが改善され十分な母数が確保できたら → Q2（CVRは高いか？）へ進む
Phase B: CVLPSs十分 & CVR高い → SEOでSs（セッション数）をさらに拡大
B
CVRが健全な状態 → Ssを増やせば売上が伸びる最も効率的なフェーズ
CVLPへの流入もありCVRも良好。この状態ではSEOでの流入拡大が売上に直結する。
▼ 確認ポイントと施策
1
既存KWの順位改善余地
確認: 5〜20位のKW群 「あと少しで1ページ目」のKWを洗い出し、伸びしろの大きいものを特定
確認: CTR（クリック率）の異常値 順位の割にCTRが低いKW = title/descriptionの改善余地あり

施策

  • SEO5〜20位KWの対象ページのコンテンツ強化（情報量・独自性の追加）
  • SEOCTRが低いKWのtitle / meta descriptionリライト
  • SEO対象ページへの内部リンク増強（関連ページ/パンくず/フッター）
▼
2
新規KW領域の開拓
確認: 競合がランクインして自社が取れていないKW Ahrefs/GSCで競合とのKWギャップを分析
確認: 未作成カテゴリ / サブカテゴリ 需要があるのにページが存在しないカテゴリがないか

施策

  • SEOKWギャップ分析に基づく新規カテゴリページ作成
  • SEO季節・トレンド系KWに対応する特集LP作成
  • SEOロングテール対応のガイド記事・比較コンテンツ追加
  • SEO商品詳細ページの独自説明文追加（ユニークコンテンツ化）
▼
3
被リンク / ドメイン評価の強化
確認: DR（ドメインレーティング）の競合比較 競合とのDR差が大きい場合、順位の天井がある可能性
確認: 被リンク獲得のトレンド 新規被リンクが増加傾向か減少傾向か

施策

  • SEOリンクされやすいコンテンツ資産の作成（調査データ・ツール等）
  • SEOPR / メディア露出によるナチュラルリンク獲得
  • SEO壊れた被リンクの修復（リダイレクト設定）
Ss拡大に伴いCVRが低下し始めたら → Phase C（歩留ボトルネック特定）へ移行
Phase C: CVLPSs十分 & CVR低い → ファネルの歩留ボトルネックを特定
C
CVLPに流入はあるのにCVしない → ファネルのどこで落ちているか特定する
4段階のファネル（一覧→詳細→カート→決済）の各遷移率を計測し、最もドロップが大きい箇所から改善する。
一覧→詳細 遷移率
詳細→カート 投入率
カート→決済開始 進行率
決済開始→完了 完了率
▼ ファネル別の確認と施策
1
一覧ページ → 商品詳細ページ（遷移率が低い場合）
確認: 商品カードの情報量 サムネイル・価格・在庫が一覧上で見えているか
確認: フィルタ/ソートの使いやすさ フィルタ利用率が低い or フィルタ後の離脱が多いか
確認: ページ送りでの離脱 2ページ目以降で急激に離脱していないか
確認: 検索結果の精度 サイト内検索結果が的外れで離脱を招いていないか

施策

  • CVR商品カードに価格・在庫バッジを表示
  • CVRサムネイル画像の品質向上（統一的な撮影 / 白背景）
  • CVRフィルタUIの改善（左カラム固定・適用中条件の明示）
  • CVR「もっと見る」ボタン / 無限スクロールの導入検討
  • CVRサイト内検索のサジェスト・表記ゆれ対応の強化
▼
2
商品詳細ページ → カート投入（投入率が低い場合）
確認: CTAの視認性 「カートに入れる」ボタンがATF（スクロールせず見える位置）にあるか
確認: 商品画像の枚数・品質 複数アングル・使用シーン・ズーム機能があるか（目安5枚以上）
確認: 購買判断に必要な情報 送料・配送日目安・返品条件が商品ページ内で明示されているか
確認: バリエーション選択UI サイズ/カラー選択がスムーズか、在庫切れの表示は適切か

施策

  • CVRCTAボタンの色・サイズ・配置の最適化（ATF固定 or 追従型）
  • CVR商品画像の追加（使用シーン・サイズ感・動画）
  • CVR送料・配送日・返品ポリシーを商品ページ内に明示
  • CVRビジュアルスウォッチ（色見本）/ 在庫連動表示
  • CVR「よくある質問」セクションの追加
▼
3
カート → 決済開始（カート離脱率が高い場合）
確認: カート内の追加コスト カートで初めて送料・手数料が表示されてショックを与えていないか
確認: 会員登録の壁 決済に進むのに会員登録が必須になっていないか
確認: カート内のアップセル過多 レコメンドが多すぎて離脱を招いていないか

施策

  • CVR送料を商品ページの段階で明示（カートでのサプライズ防止）
  • CVRゲスト購入（会員登録なし）の導線を設置
  • CVRカート離脱リマインドメール / LINE通知の設定
  • CVRカート内レコメンドの適正化（1〜2点に絞る）
  • CVR「あとで買う」/ お気に入り保存機能の強化
▼
4
決済開始 → 決済完了（完了率が低い場合）
確認: フォーム入力のハードル 入力項目数、エラー表示のわかりやすさ、住所自動入力の有無
確認: 決済手段の種類 クレカ・コンビニ・後払い・QR決済・Apple Pay等が揃っているか
確認: セキュリティへの不安 SSL表示・決済バッジ・プライバシーポリシーが明示されているか

施策

  • CVREFO（入力フォーム最適化）: 項目数削減・ステップ分割・リアルタイムバリデーション
  • CVR住所自動入力 / クレカ情報のオートフィル対応
  • CVR決済手段の拡充（Amazon Pay / 後払い / QR決済）
  • CVRセキュリティバッジ・返品保証の表示
  • CVR決済ステップのプログレスバー表示
歩留改善でCVRが目標値に達したら → Phase B（Ss拡大）に移行し、売上最大化へ
Phase D: 施策の優先順位付け → 実行管理
!
各Phaseで特定された施策をインパクト × 工数で優先順位付け
Phase A〜Cで洗い出した施策を一覧化し、「インパクト（1-5） × （6 - 工数）」の優先スコアで並び替える。
Excelテンプレートの「施策優先度」シートに記録し、担当・期限・KPIを設定して実行管理へ移行。
高インパクト × 低工数
→ 即実行（Quick Win）
高インパクト × 高工数
→ 計画的に実行（Strategic）
低インパクト × 低工数
→ 余裕があれば実行
低インパクト × 高工数
→ 見送り or 再検討
Phase間の循環:
Phase A（CVLP改善） → Ssが増えたら → Phase B or C へ
Phase B（Ss拡大） → CVRが低下したら → Phase C へ
Phase C（歩留改善） → CVRが回復したら → Phase B へ
このサイクルを回し続けることで継続的な売上成長を実現する。
